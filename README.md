# BlokPOST - Система управленческой отчетности

Веб-приложение для управленческой отчетности в сети магазинов спецодежды «БлокПОСТ». Объединяет разрозненные данные из Excel в единую версию правды.

## Архитектура

- **Фронтенд**: Next.js с TypeScript, Tailwind CSS
- **Бэкенд**: Next.js API Routes
- **База данных**: PostgreSQL с Prisma ORM
- **Авторизация**: NextAuth.js с ролевой моделью
- **ETL**: Обработка Excel файлов с помощью xlsx

## Сущности

- Магазины
- Товары (с категориями)
- Продажи
- Остатки
- Справочник KPI
- Пользователи с ролями: Собственник, Аналитик, Директор магазина

## Запуск

1. Установите зависимости: `npm install`
2. Настройте базу данных PostgreSQL и обновите `.env`
3. Запустите миграции: `npx prisma migrate dev`
4. Запустите сервер: `npm run dev`

## Разработка

- Дашборд собственника: Полный доступ к метрикам
- Ролевая авторизация
- Автоматический расчет KPI
- Импорт данных из Excel

## API для загрузки Excel с продажами

Добавлен маршрут `POST /api/upload-sales`.

Ожидаемый файл должен содержать колонки:
- `store_id`
- `receipt_number`
- `product_sku`
- `quantity`
- `unit_price`
- `date`

Если магазин или товар не найдены в справочнике, файл отклоняется с ошибкой для контроля качества данных.

### Как использовать

1. Отправьте `multipart/form-data` с полем `file`.
2. Если данные валидны, записи сохраняются в таблицу `Sale`.
3. В случае несоответствия справочникам API вернет `400` и список проблем.
