# BlokPOST - Система управленческой отчетности

Веб-приложение для управленческой отчетности в сети магазинов спецодежды «БлокПОСТ». Объединяет разрозненные данные из Excel в единую версию правды.

## 🚀 Быстрый запуск демо версии

### Предварительные требования

- **Node.js** версии 18 или выше
- **npm** или **yarn**

### Шаг 1: Клонирование и установка

```bash
# Клонируйте репозиторий
git clone https://github.com/SunnyS8/Blok_post.git
cd Blok_post

# Установите зависимости
npm install
```

### Шаг 2: Настройка переменных окружения

```bash
# Скопируйте файл с примером
cp .env.example .env

# Или создайте .env файл с содержимым:
echo 'DATABASE_URL="file:./dev.db"
NEXTAUTH_URL="http://localhost:3000"
NEXTAUTH_SECRET="demo-secret-key-for-development-only"
JWT_SECRET="demo-jwt-secret-for-development-only"' > .env
```

### Шаг 3: Инициализация базы данных и демо данных

```bash
# Генерация Prisma клиента
npm run db:generate

# Создание и применение миграций
npm run db:migrate

# Заполнение демо данными
npm run db:seed
```

### Шаг 4: Запуск приложения

```bash
# Запуск в режиме разработки
npm run dev

# Или однокомандный запуск демо версии (сброс БД + демо данные + запуск)
npm run demo:start
```

Приложение будет доступно по адресу: **http://localhost:3000**

## 🏗️ Архитектура

- **Фронтенд**: Next.js с TypeScript, Tailwind CSS
- **Бэкенд**: Next.js API Routes
- **База данных**: SQLite с Prisma ORM
- **Авторизация**: NextAuth.js с ролевой моделью
- **ETL**: Обработка Excel файлов с помощью xlsx

## 🧪 Supabase Prototype

Для прототипа интеграции добавлены страницы:

- `/supabase-demo` — проверка подключения к Supabase, авторизация и просмотр данных
- `/supabase-upload` — импорт Excel-файла в Supabase как заказы и позиции

Чтобы использовать прототип, настройте в `.env.local` следующие переменные:

```env
NEXT_PUBLIC_SUPABASE_URL="https://your-project.supabase.co"
NEXT_PUBLIC_SUPABASE_ANON_KEY="your-anon-key"
SUPABASE_SERVICE_ROLE_KEY="your-service-role-key"
```

Для создания таблиц используйте файл `SUPABASE_SCHEMA.md`.

Проект также поддерживает таблицу `profiles` для хранения ролей пользователей. Вы можете создавать и обновлять профиль через API:

- `POST /api/supabase/profiles` — создать или обновить роль пользователя

После настройки откройте:

```text
http://localhost:3000/supabase-demo
http://localhost:3000/supabase-upload
```

4. Проверьте логи в терминале на ошибки

## API для загрузки Excel с продажами

Добавлен маршрут `POST /api/upload-sales`.

Ожидаемый файл должен содержать колонки:
- `store_id`
- `receipt_number`
- `product_sku`
- `quantity`
- `unit_price`
- `date`

Если магазин или товар не найдены в справочнике, файл отклоняется с ошибкой для контроля качества данных.

### Как использовать

1. Отправьте `multipart/form-data` с полем `file`.
2. Если данные валидны, записи сохраняются в таблицу `Sale`.
3. В случае несоответствия справочникам API вернет `400` и список проблем.
